2+^h|*ADL_CODE_DEFINITIONS*

2+a|Definitions relating to the internal code system of archetypes.

2+h|*Constants*

|*Id_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"id"` [1..1]
a|String leader of ‘identifier’ codes, i.e. codes used to identify archteype nodes.

|*Value_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"at"` [1..1]
a|String leader of ‘value’ codes, i.e. codes used to identify codes values, including value set members.

|*Value_set_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"ac"` [1..1]
a|String leader of ‘value set’ codes, i.e. codes used to identify value sets.

|*Specialisation_separator*: `xref:BASE:foundation_types:primitive_types.adoc#_character_class[Character]{nbsp}={nbsp}'.'` [1..1]
a|Character used to separate numeric parts of codes belonging to different specialisation levels.

|*Code_regex_pattern*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"(0&#124;[1-9][0-9]&#42;)(\.(0&#124;[1-9][0-9]&#42;))&#42;"` [1..1]
a|Regex used to define the legal numeric part of any archetype code. Corresponds to the simple pattern of dotted numbers, as used in typical multi-level numbering schemes.

|*Root_code_regex_pattern*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"^id1(\.1)&#42;$"` [1..1]
a|Regex pattern of the root id code of any archetype. Corresponds to codes of the form id1, id1.1, id1.1.1 etc..

|*Primitive_node_id*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"id9999"` [1..1]
a|Code id used for C_PRIMITIVE_OBJECT nodes on creation.
