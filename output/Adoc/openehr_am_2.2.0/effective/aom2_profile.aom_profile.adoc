2+^h|*AOM_PROFILE*

2+a|Profile of common settings relating to use of reference model(s) and terminology for a given archetype developing organisation.

2+h|*Attributes*

|*aom_rm_type_substitutions*: `xref:BASE:foundation_types:structure_types.adoc#_hash_class[Hash]<xref:BASE:foundation_types:primitive_types.adoc#_string_class[String],xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]>` [0..1]
a|Allowed type substitutions: Actual RM type names keyed by AOM built-in types which can subsitute for them in an archetype. E.g. <value = "String", key = "ISO8601_DATE"> means that if RM property TYPE.some_property is of type String, an ISO8601_DATE is allowed at that position in the archetype.

|*aom_lifecycle_mappings*: `xref:BASE:foundation_types:structure_types.adoc#_hash_class[Hash]<xref:BASE:foundation_types:primitive_types.adoc#_string_class[String],xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]>` [0..1]
a|List of mappings of lifecycle state names used in archetypes to AOM lifecycle state names. value = AOM lifecycle state; key = source lifecycle state.

|*profile_name*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]` [1..1]
a|Name of this profile, usually based on the publisher it pertains to e.g. "openEHR", "cdisc", etc.

|*aom_rm_type_mappings*: `xref:BASE:foundation_types:structure_types.adoc#_hash_class[Hash]<xref:BASE:foundation_types:primitive_types.adoc#_string_class[String],xref:AM:aom2_profile_package.adoc#_aom_type_mapping_class[AOM_TYPE_MAPPING]>` [0..1]
a|Mappings from AOM built-in types to actual types in RM: whenever the type name is encountered in an archetype, it is mapped to a specific RM type.

|*rm_primitive_type_equivalences*: `xref:BASE:foundation_types:structure_types.adoc#_hash_class[Hash]<xref:BASE:foundation_types:primitive_types.adoc#_string_class[String],xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]>` [0..1]
a|Equivalences of RM primitive types to in-built set of primitive types. Used to determine which AOM C_PRIMITIVE_OBJECT descendant is used for a primitive type. Typical entries:

* value	key
* "Real"	"Double"
* "Integer"	"Integer64"
