2+^h|*ADL_CODE_DEFINITIONS*

2+a|Definitions relating to the internal code system of archetypes.

NOTE: ADL2 was initially released with code system using id-codes (for nodes), at-codes (for values only) and ac-codes (for value sets). As this proved to hinder its widespread use in the openEHR community (due to its impact on existing openEHR RM data), a code system matching the one used in ADL1.4 was introduced as the openEHR primary code system.

openEHR systems must support at least the at-coded ADL2 archetypes.

2+h|*Constants*

|*At_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"at"` [1..1]
a|String leader of ‘identifier’ codes, i.e. codes used to identify at-coded archetype nodes.

|*Id_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"id"` [1..1]
a|String leader of ‘identifier’ codes, i.e. codes used to identify id-coded archetype nodes.

|*Value_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"at"` [1..1]
a|String leader of ‘value’ codes, i.e. codes used to identify codes values, including value set members.

|*Value_set_code_leader*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"ac"` [1..1]
a|String leader of ‘value set’ codes, i.e. codes used to identify value sets.

|*Specialisation_separator*: `xref:BASE:foundation_types:primitive_types.adoc#_character_class[Character]{nbsp}={nbsp}'.'` [1..1]
a|Character used to separate numeric parts of codes belonging to different specialisation levels.

|*Code_regex_pattern*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"(0&#124;[1-9][0-9]&#42;)(\.(0&#124;[1-9][0-9]&#42;))&#42;"` [1..1]
a|Regex used to define the legal numeric part of any archetype code. Corresponds to the simple pattern of dotted numbers, as used in typical multi-level numbering schemes.

|*Root_code_regex_pattern*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"^(id1&#124;at0000)(\.1)&#42;$"` [1..1]
a|Regex pattern of the root code of any archetype. Corresponds to at-codes of the form `at0000`, `at0000.1`, `at0000.1.1` etc, and id-codes of the form `id1`, `id1.1`, `id1.1.1` etc. For at-coded ADL2 numbering starts with zero (i.e. `at0000`) while the id-coded ADL2 numbering starts with one (i.e. `id1`).

|*Primitive_node_id*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]{nbsp}={nbsp}"at9999"` [1..1]
a|Code id used for `C_PRIMITIVE_OBJECT` nodes on creation.

NOTE: For id-coded archetypes this is `"id9999"`.
