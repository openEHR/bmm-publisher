2+^h|*EXPR_LEAF*

2+a|Expression tree leaf item representing one of:

* a manifest constant of any primitive type;
* a path referring to a value in the archetype;
* a constraint;
* a variable reference.

2+|Inherits: xref:AM:AOM1.4:archetype_package.adoc#_expr_item_class[EXPR_ITEM]

2+h|*Attributes*

|*type*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]` [1..1]
a|Type name of this item in the mathematical sense. For leaf nodes, must be the name of a primitive type, or else a reference model type. The type for any relational or boolean operator will be “Boolean”, while the type for any arithmetic operator, will be “Real” or “Integer”. +
_Inherited from xref:AM:AOM1.4:archetype_package.adoc#_expr_item_class[EXPR_ITEM]_

|*reference_type*: `xref:BASE:foundation_types:primitive_types.adoc#_string_class[String]` [1..1]
a|Type of reference: “constant”, “attribute”, “function”, “constraint”. The first three are used to indicate the referencing mechanism for an operand. The last is used to indicate a constraint operand, as happens in the case of the right-hand operand of the ‘matches’ operator.

|*item*: `xref:BASE:foundation_types:overview.adoc#_any_class[Any]` [1..1]
a|The value referred to; a manifest constant, an attribute path (in the form of a String), or for the right-hand side of a ‘matches’ node, a constraint, often a C_PRIMITIVE_OBJECT.
